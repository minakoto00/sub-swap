**2. Two helper functions:**

`read_source(rel_path: &str) -> String` — joins `env!("CARGO_MANIFEST_DIR")` with `rel_path`, reads file with `std::fs::read_to_string`, panics with clear message if file not found.

`assert_no_crate_import(file: &str, source: &str, forbidden_modules: &[&str])` — extracts all lines starting with `use crate::` (after trimming whitespace), then for each line checks if it contains `crate::{forbidden}` for any forbidden module. On match, panics with the exact three-part format per D-08:
```
VIOLATION: Layer boundary — {file} must not import {forbidden}
FOUND: {the full use crate:: line}
HOW TO FIX: Remove the `use crate::{forbidden}` import from {file}. {file} is in a lower layer than {forbidden}. If this logic is needed, move it to the orchestration layer (cli.rs or tui/) or restructure the dependency.
```

- Call `read_source("src/{path}")` to get file contents
