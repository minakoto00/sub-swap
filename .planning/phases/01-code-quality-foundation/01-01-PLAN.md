Fix every clippy violation listed in RESEARCH.md section "Exact Clippy Violations to Fix". The complete list:

**Dead code fixes (6 warnings):**

1. `src/crypto/keychain.rs` — Gate `MockKeyStore` struct, its `impl MockKeyStore`, its `Default` impl, and its `impl KeyStore for MockKeyStore` with `#[cfg(test)]`. The `has_key()` method on the `KeyStore` trait itself must stay (used by `OsKeyStore` in production in `cli.rs`). The trait method `has_key` will not trigger `dead_code` because `OsKeyStore::has_key()` is called in production code.

2. `src/guard.rs` — Gate `MockGuard` struct, its `impl MockGuard`, and its `impl CodexGuard for MockGuard` with `#[cfg(test)]`. These are test-only constructs.

3. `src/error.rs` — The `NotInitialized` variant is never constructed. Remove it entirely (it can be re-added if needed). Also remove its `Display` match arm (`Self::NotInitialized => write!(f, "sub-swap is not initialized. Run sub-swap to set up.")`). Alternatively, if removal feels too aggressive, add `#[allow(dead_code)]` to just that variant with a comment: `// Reserved for future initialization check`.

**Pedantic violation fixes (all mechanical, per D-02 — fix real issues):**

4. `src/cli.rs:21` — `if_not_else`: Find the `if !<condition> { ... } else { ... }` pattern and invert it. The block at line 21 area should become `if <condition> { <else-body> } else { <if-body> }`.

5. `src/cli.rs:106` — `single_match_else`: Replace `match from { Some(source_path) => ..., None => ... }` with `if let Some(source_path) = from { ... } else { ... }`.

6. `src/crypto/keychain.rs:8` — `format_collect`: The `encode_key` function uses `key.iter().map(|b| format!("{b:02x}")).collect()`. Replace with a fold approach: `key.iter().fold(String::with_capacity(64), |mut s, b| { use std::fmt::Write; let _ = write!(s, "{b:02x}"); s })`. Or simply use `key.iter().map(|b| format!("{b:02x}")).collect::<String>()` with an `#[allow(clippy::format_collect)]` if the fold is less readable — but prefer the fold fix since clippy recommends it.

7. `src/error.rs:37` — `redundant_closure_for_method_calls`: Change `pids.iter().map(|p| p.to_string()).collect()` to `pids.iter().map(ToString::to_string).collect()`.

8. `src/profile/mod.rs:96` — `redundant_closure_for_method_calls`: In the `names()` method, change `.map(|s| s.as_str())` to `.map(String::as_str)`.

9. `src/tui/mod.rs` — Multiple `single_match_else` violations at lines 185, 222, 272, 361, 417: Each is a `match` with two arms (one specific, one wildcard). Convert each to `if let`. These are in `handle_confirm_switch`, `handle_force_switch`, `handle_confirm_delete`, `handle_input_name` (inside match on `pending_action`), and `handle_input_note`.

10. `src/tui/mod.rs` — Multiple `redundant_closure_for_method_calls` at lines 254, 292, 350, 381: Change `.map(|s| s.to_string())` to `.map(ToString::to_string)` in the `store.index.names().into_iter()` chains.

11. `src/tui/mod.rs` — Multiple `assigning_clones` at lines 256, 294, 352, 383: Change `state.active_profile = store.index.active_profile.clone()` to `state.active_profile.clone_from(&store.index.active_profile)`.

12. `src/tui/widgets.rs:38` — `redundant_closure_for_method_calls`: In `from_index`, change `.map(|s| s.to_string())` to `.map(ToString::to_string)`.

13. `src/tui/widgets.rs:54` — `redundant_closure_for_method_calls`: In `selected_name`, change `.map(|s| s.as_str())` to `.map(String::as_str)`.

14. `src/tui/wizard.rs` — `uninlined_format_args` at lines 48, 118, 142, 143, 164: Change `format!("... {}", var)` patterns to `format!("... {var}")` using inline format arguments. Specific instances:
    - Line 48: `println!("Profile '{}' created successfully.", name)` → `println!("Profile '{name}' created successfully.")`
    - Line 118: `print!("{} {} ", question, choices)` → `print!("{question} {choices} ")`
    - Line 142: `print!("{} [{}]: ", question, d)` → `print!("{question} [{d}]: ")`
    - Line 143: `print!("{}: ", question)` → `print!("{question}: ")`
    - Line 164: `print!("{}: ", question)` → `print!("{question}: ")`

15. `src/tui/wizard.rs:133` — `match_same_arms`: The match at line 130-135 has identical arms for `""` and `_`. Collapse `"" | _` into a single wildcard `_` arm: `_ => default_yes`.
